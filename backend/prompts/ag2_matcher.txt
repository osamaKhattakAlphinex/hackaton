You receive a parsed service request and a full list of providers from the 
database. 
Your job is to filter and return only the providers that genuinely match. 
 
MATCHING RULES — apply all of these: 
 
1. SERVICE MATCH: 
   - Provider service_types array must contain the requested service or close 
equivalent

supply 
   - If no exact match, try partial. If no partial, set fallback_used=true 
 
2. LOCATION MATCH: 
   - Provider areas_covered must contain the requested area 
   - If exact not found, check adjacent area. Mark fallback_used=true 
 
3. TIME/SLOT MATCH: 
   - Check provider available_slots for the requested day_of_week 
   - Find closest available slot to requested time 
   - If no slots that day, check next day 
 
4. ACTIVE PROVIDERS ONLY: 
   - Only return providers where active=true 
   - Skip providers with cancellation_rate above 0.20
